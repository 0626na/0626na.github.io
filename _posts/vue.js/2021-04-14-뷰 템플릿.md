## 뷰 템플릿
뷰에서 템플릿(template)은 HTML, CSS등의 마크업 속성과 뷰 인스턴스에서 정의한 데이터 및 로직들을 연결하여 유저가 브라우저에서 볼수있는 HTML 형태로 변환해주는 속성입니다.

```javscript
<script>
  new Vue({
    template: '<p>{{message}}</p>'
  })
</script>
```

템플릿은 뷰 인스턴스 내에 template 속성을 정의해서 사용하는 방법이 있습니다. 이는 ES5 방식으로 실무에서는 잘 쓰이지 않는 방식이다. 
템플릿 속성은 유저는 볼수 없지만 내부에서 template 속성에 정의한 html태그와 뷰 데이터를 가상 DOM 기반의 render 함수로 변환합니다. 
이 render 함수가 최종적으로 유저에게 보이는 화면을 만듭니다. 그리고 렌더링 과정중에 뷰 반응성을 화면에 부여합니다.

- template 속성을 사용하고 안하고의 차이
template 속성을 쓰지 않고 바로 태그를 작성하면 뷰는 먼저 화면에 태그들을 출력하고 인스턴스가 생성되고 나서 뷰 데이터들을 화면에 나타냅니다.
template 속성을 사용하면 인스턴스 생성전까지 아무것도 표시되지 않다가 인스턴스가 생성되고 화면을 출력합니다.

- template을 안쓰는경우
```javascript
  <div id="app">
    <h3>{{message}}</h3>
  </div>
  
  <script>          // template 속성을 쓰지않으면 태그가 실행가 동시에 화면에 출력, 인스턴스 생성 이후, 데이터를 변환
    new Vue({
      el: "#app",
      data: {
        message: "hello vue"
      }
    });
  </script>
```
- template을 쓰는경우
```javascript
  <div id="app">
  </div>
  
  <script>          // template 속성을 쓰면 인스턴스 생성 이후, 화면이 출력
    new Vue({
      el: "#app",
      data: {
        message: "hello vue"
      },
      template: '<h3>{{message}}</h3>';
    });
  </script>
```

## 템플릿에서의 뷰의 속성과 문법
### 데이터 바인딩
HTML 화면요소와 뷰 인스턴스의 데이터의 상호연결을 말합니다. {{}} 문법과 v-bind 속성을 사용합니다.
- 콧수염 괄호
뷰에서는 {{}}를 콧수염 괄호라고 부른다. 콧수염 모양과 비슷하여 콧수염 괄호라고 부릅니다. html 태그에 뷰 데이터를 연결하는 기본적인 삽입 방식입니다.
이 콧수염 괄호는 뷰 뿐만아닌 대부분의 프론트엔드 프레임워크에서 사용하는 템플릿 문법입니다. 콧수염 괄호안의 데이터는 뷰 반응성에 의해 데이터가 변경되면 자동으로 화면의 데이터도 변경됩니다.
- v-bind
아이디, 클래스, 스타일등의 HTML 속성값을 뷰 데이터와 연결할때 사용하는 뷰 속성입니다.

```javascript
  <div id="app">
    <p v-bind:id="idVueData">아이디</p>
    <p v-bind:class="classVueData">클래스</p>  // bind에 태그의 속성과 뷰 데이터를 서로 연결해준다. bind를 사용하면 이렇게 뷰 데어터와 태그 속성간의 연결이 가능하다.
  </div>
  
<script>
  new Vue({
  el: "#app",
  data: {
    idVueData: 10,
    classVueData: "container",
  },
});
</script>
```
v-bind는 :로 축약해서 사용할수 있습니다. v-bind:id="idVueData" -> :id="idVueData"

### 자바스크립트 표현식
템플릿에는 {{}} 안에 자바스크립트 표현식으로 작성이 가능하다. 데이터에 자바스크립트 문법 및 연산 사용이 가능하다.

```javascript
  <div id="app">
    <p>{{message}}</p>
    <p>{{message + !!!}}</p>
    <p>{{message.split(' ')}}</p>  // 자바스크립트의 내장 함수 및 자바스크립트 같이 문자열 연결등의 표현식 사용이 가능
  </div>
  
<script>
  new Vue({
  el: "#app",
  data: {
    message: "Hello nills",
  },
});
</script>
```
자바스크립트 표현식을 템플릿 내에 사용할때는 자바스크립트 선언문과 분기문은 사용이 불가능합니다. 또한 복잡한 연산은 뷰 인스턴스 내에서 처리하고 화면에는 단순하게 표현해야 합니다.
뷰에서는 화면의 코드의 가독성을 높이기 위해서 되도록 태그에는 출력될 값만 나타내고 데이터 연산은 기본적으로 인스턴스 내에서 하도록 권하고 있습니다.

### 뷰 디렉티브
뷰 디렉티브는 html 태그안에 v- 접두사를 가지는 모든 속성을 말합니다. html 속성과 같이 태그 내에 작성합니다.
디렉티브는 뷰가 화면의 요소를 더 쉽게 조작하기 위해 사용하는 기능입니다. 이러한 디렉티브를 사용하여 화면의 태그들을 직접 제어할 필요없이 전부 뷰에서 제어가 가능하게 합니다.

- 주로사용하는 디렉티브
  1.  **v-if**: 뷰 데이터의 값이 true/false 에 따라 태그를 화면에 표시하거나 하지않거나 정합니다. 프로그래밍의 if문과 같은 역할을 합니다.
  2.  **v-for**: 지정한 뷰 데이터 갯수만큼 태그를 만들어내어 반복 출력합니다. 이때, v-for을 정의할때는 v-bind:key 속성을 같이 명시해야 합니다.
  3.  **v-show**: v-if와 마찬가지로 true/false에 따라 화면에 표시하거나 하지않거나 하는 속성이다. v-if와의 차이는 v-if의 경우는 거짓일때 화면에 태그자체가 존재하지 않는다. 
      하지만 v-show에서는 태그에 display:none css 속성을 정의하여 실제 태그는 존재하지만 보이지는 않게 합니다.
  4.  **v-bind**: html 태그 속성에 뷰 데이터를 연결하는 속성입니다. :로 약어로 사용이 가능합니다.
  5.  **v-on**: 화면 요소의 이벤트를 감지하여 처리하는 속성입니다. v-on:click 과 같이 이벤트에 특정 메서드를 연결하여 이벤트를 처리하게 합니다. v-on은 @으로 약어로 사용이 가능합니다.
  6.  **v-model**: html 요소와 뷰 데이터를 연결할때 사용하는 속성입니다. v-model 로 연결한 데이터는 html 요소와 뷰 데이터 양방향 바인딩이 이루어져 실시간으로 값이 양쪽에 적용됩니다. 
      주로 form에서 사용하는 속성으로 input, select, textarea 태그에서만 사용이 가능합니다.

## 고급 템플릿 기법
실제 애플리케이션을 개발할때 유용한 속성들이다.

### computed 속성
데이터를 연산하고 가공하는 등의 복잡한 연산을 정의하는 속성입니다. computed 속성의 첫번째 장점은 data 속성값이 변함에 따라 자동으로 재연산 한다는 것입니다. 두번째로는 캐싱입니다.
데이터 연산을 템플릿에 작성하면 템플릿을 불러올때 마다 매번 연산이 이루어집니다. 이는 성능에 영향을 줄수있습니다. 캐싱은 데이터가 변하지 않는 한 연산한 값을 저장하는 방식입니다.
데이터가 바뀌지 않는이상은 화면을 불러올때 마다 이미 계산되어 있는 캐싱값을 주어 바로 출력이 가능하게 합니다.

- computed 속성과 methods 속성의 차이
methods 속성은 호출할때만 속성 내부의 메서드들을 실행하고 computed에 정의된 것들은 데이터가 변경이 있을때 마다 자동으로 실행된다는 점입니다. 즉, 자동으로 변환할지 수동으로 변환 할지의 차이입니다.

### watch 속성
데이터 변화를 감지하여 자동으로 속성에 정의된 로직을 수행합니다. computed 속성과 비슷하지만 watch는 비동기처리로 작동합니다.
watch 속성은 비동기로 작동하기에 watch 속성을 자주 이용하면 프로그램이 에러가 나기 쉽고 디버깅이 어려워집니다. 그렇기에 뷰에서는 watch 속성을 사용하는것을 추천하지 않습니다.
